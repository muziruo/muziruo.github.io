# Runtime 简介

每个 iOS 程序员应该都或多或少听过 Runtime 这个词，Runtime 是 iOS 应用的运行的核心机制，我们所用到的类、库，以及方法调用都和 Runtime 相关。

# Runtime 基础属性

方法结构体，用于描述类中的方法。其中包括方法名、方法类型以及指向方法实现的指针 IMP 。

实例变量结构体，用于描述类的实例变量。包含变量名、变量类型、 Ivar 偏移量。

# 类结构体 objc_class

  • isa；指向所属类的指针。一个实例对象的 isa 指针指向他所属的类，一个类的 isa 指针指向他所属元类，元类的 isa 指针指向根元类，根元类的 isa 指针指向自身。如图：
  • super_class；所属父类。
  • name；类名。
  • version；版本。
  • info；供运行期使用的一些标志位。
  • instance_size；生成的实例对象大小。
  • ivars；类的实例对象。
  • methodLists；类中的方法列表。
  • cache；最近使用的方法调用，用于优化方法调用。在消息传递的过程中，会优先查找 cache 中保留的方法，如果恰好为需要调用的方法，则直接利用 IMP 寻找到方法的具体实现。
  • protocols；类中的协议列表。

# 消息传递

消息传递，即我们常说的方法调用。我们在以上这一些内容的基础上来探索一下 iOS 中方法调用的过程。我们先看一下 objc.h 中关于对象的定义。

我们可以看到里面仅有一个 isa 指针，而 isa 指针我们在上面已经见到过了，类的 isa 指针指向元类，此处的 isa 指针指向类，类比一下我们可以发现，其实类也是一个对象，是元类的实例对象，这儿我们先不说类与元类的关系，我们先了解此处对象的结构。

以上是我们常见的方法调用语句，我们来追溯一下方法调用的过程：

  1. 首先 obj 对象要通过 isa 指针来获取自己所属的类；
  2. 找到对应的类后，我们可以看看上一节关于类的结构体介绍，类中包含方法列表、方法缓存以及指向父类的指针 super_class，可知此时已经可以开始对方法进行寻找，首先是寻找方法缓存 cache，如果方法缓存中有所需要调用的方法，则就找到了对应方法，否则再查找方法列表 methodLists；
  3. 如果以上两个地方未找到对应的方法，则会根据指向父类的指针找到父类，在父类中同样会在方法缓存以及方法列表中查找；
  4. 按以上流程不断回溯，一旦找到对应的方法，则会将对应方法的 IMP 指针存放到当前方法所属类中的 cache 方法缓存中，方便下次寻找。如果到最终的基类，如果依旧没有找到方法，则会启动 消息转发 机制。
  5. 如果最后消息转发依旧未找到方法，则会抛出异常， unrecognized selector sent to …

以上这些就是方法调用的基本过程。

# 消息转发

上一标题中我们在消息传递的过程中提到了消息转发，现在我们就来说一说消息转发。消息转发总共有三个要点 —— 动态方法解析、备用接收者以及完整消息转发。
