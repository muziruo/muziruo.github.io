计算机网络

发表于2021-03-19|更新于2022-04-17
|阅读量:

概述

计算机网络是由若干结点以及连接这些结点的链路组成，其中结点可以是计算机、集线器、交换机以及路由器等。

互联网

互联网发展三个阶段：

  1. ARPANET，美国军方创造的第一个分组交换网。
  2. NSFNET，国家科学基金网，是一个三级计算机网络，分为主干网、地区网以及企业网。
  3. 开始形成多层次的ISP结构的互联网，ISP即互联网服务提供商。一般来说层级有主干ISP、地区ISP、本地ISP最后到校园网(企业网)或者是主机。

互联网的两大结构：

  1. 边缘部分；由所有连接在互联网上的主机组成。这部分是用户直接使用的，用来进行通信和资源共享。
  2. 核心部分：由大量网络和连接这些网络的路由器组成，这部分是为边缘部分提供服务的(提供了连通性和交换)。

三种交换方式

  1. 电路交换；整个报文的比特流从源头直接到终点，类似于在管道中传送。
  2. 报文交换；整个报文先传送到相邻结点，全部存储下来后查找转发表，转发到下一个结点。
  3. 分组交换；单个分组传送到相邻结点，存储下来后查找转发表，转发到下一结点。与报文交换类似，但是不同的点在于分组通常是报文的一部分，可以看作是把报文进行了切割。

计算机网络的性能指标

速率

也称为数据率，表示每秒传输的二进制信息量。

带宽
  1. 传统意义上带宽是表示信号所具有频率宽度，一般是 XX HZ～XX HZ，以赫兹(HZ)为单位
  2. 在计算机网络中的带宽，表示在单位时间内网络中的某信道所能通过的最高数据率，和数据率的单位一致。
吞吐量

表示单位时间内通过某个网络的实际数据量。

时延

指数据从网络的一端到另一端所需要的时间。

  1. 发送时延；发送时延是主机或路由器发送数据帧所需要的时间，也就是从发送数据帧的第一个bit算起，到发送完最后一个bit的时间，计算公式为 发送时延 = 数据帧长度（bit）/ 发送速率 （bit/s）。
  2. 传播时延；电磁波在信道中传播一定的距离所需要花费的时间。计算公式为 传播时延 = 信道长度 / 电磁波在信道上传播的速率 。
  3. 处理时延；主机或路由器在收到分组后是需要花费一定的时间进行处理转发的。这部分时间就是处理时延。
  4. 排队时延；分组在经过网络传输的时候，是需要经过很多路由器的，但分组在进入路由器后并不是就能立即处理转发出去，而是需要在输入队列中进行排队等待处理，这部分时间就是排队时延。

总时延 = 以上几个时延加起来的总和。

时延带宽积

传播时延与带宽的乘积即为时延带宽积。表示填满整个传输管道需要的比特数。

往返时间RTT

数据发送完毕后到接受到另一端的确认信息所需要的时间。在计算有效数据率的时候，我们需要在发送时间的基础上加上RTT时间。公式为 有效数据率 = 数据长度 / （发送时间 + RTT）。

利用率

分为信道利用率和网络利用率两种。

信道利用率指某信道有百分之几的时间是被利用的。信道利用率并非越高越好，信道利用率增加的时候，信道时延也会相应增加。一般主干网络的信道利用率保持在50%左右，如果更大则需要考虑扩容。

网络利用率则是全网络的信道利用率的加权平均值。

计算机网络的非性能特征

费用、质量、标准化、可靠性、可扩展性和可升级性、易于管理和维护

计算机网络体系结构

计算机网络的各层及其协议的集合就是网络的体系结构。

协议与划分层次

分层的好处
  1. 各层之间是独立的。不同层之间不需要互相知道其他层的实现，只需要调用其他层提供的接口即可。
  2. 灵活性好。
  3. 结构上可分割开。
  4. 易于实现和维护。
  5. 能促进标准化工作。一般来说每层所需要考虑的问题有：差错控制、流量控制、分段与重装、复用和分用以及连接建立与释放。

计算机网路体系五层结构

应用层

交互的数据单元称作为报文。

运输层
  • TCP
    • 面向连接的、可靠的数据传输服务。数据传输的单位是报文段。
  • UDP
    • 无连接的，尽最大努力的数据传输服务，不保证数据传输的可靠性，传输单位是用户数据报。
网络层

将运输层的报文段或者用户数据报封装成分组或者包进行传送。

数据链路层

传输数据单位为帧。每一帧包括数据和必要的控制信息(如同步信息、地址信息、差错控制等)

物理层

传输数据单位为比特。该层所需要考虑的是使用怎样的电压来表示1和0，并且规定电缆插头的引脚有多少根以及如何连接。

OSI参考模型吧对等层次之间传送的数据单位称为该层的**协议数据单元(PDU，Protocol Data Unit)**。

实体、协议、服务和服务访问点

实体

在研究开放系统中的信息交换时，往往使用实体这一较为抽象的名词来表示任何可发送和接收信息的硬件或软件进程。一般来说就是一个软件模块。

协议

协议是控制两个对等实体或多个实体之间进行通信的规则的集合。

服务

在协议的控制下，两个对等实体间的通信使得本层能够向上一层提供服务，要实现本层的协议，还需要使用下面一层所提供的服务。

协议是水平的，同层次之间的数据传递，而服务是垂直的，由下层为上层提供接口实现。

服务访问点

同一系统中相邻两层的实体进行交互的地方，通常称为**服务访问点(SAP，Service Access Point)。这是一个比较抽象的概念，其实上就是一个逻辑接口。OSI把层与层之间交换的数据的单位称为服务数据单元SDU(Service Data Unit)**。

计算机网络的协议需要预先估计到所有不利的条件情况，不能假定一切都是正常和理想化的。

物理层

物理层的主要任务描述为确定与传输媒体的接口有关的一些特性，即：机械特性、电气特性、功能特性、过程特性。
